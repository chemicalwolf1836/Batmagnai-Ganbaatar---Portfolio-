Wander

An AI-powered travel discovery app that helps you find your perfect destination through natural conversation, guided prompts, or a single tap — with an interactive map, 3D card effects, and built-in trip planning tools.

Live Demo ↗ Source Back to Projects
Overview

Project Summary

Wander is a full-stack travel app powered by Claude AI. Users describe their ideal trip in natural language or tap through guided choice chips — the AI picks three matching destinations shown on an interactive world map, each with full cultural details, weather, food, and planning tools.

Stack

Stack & Architecture

  • Next.js 16 — App Router, server components, API routes
  • TypeScript — strict mode throughout
  • Tailwind CSS — styling system
  • Framer Motion — animations and transitions
  • Claude API — destination suggestions, packing lists, itineraries
  • Supabase — PostgreSQL database, result caching
  • Mapbox GL — interactive world map
  • OpenWeatherMap — live weather per destination
  • Vercel — deployment
AI

AI Integration

  • Discovery chat — free-text or guided chip flow; Claude determines when it has enough information to search
  • Destination suggestions — Claude returns structured JSON with coordinates, cultural theme colours, attractions, food, and practical info
  • Packing list — Claude Haiku generates a categorised list based on destination and trip length
  • Itinerary builder — day-by-day plan with morning, afternoon, evening slots and a local tip per day
  • Similar destinations — Claude suggests 3 more places with a matching vibe
  • Result caching — Supabase stores suggestions for 24 hours to reduce API calls
Features

UI & UX Features

  • 3D card tilt + image parallax — cards tilt in perspective on hover; background photo shifts opposite direction for depth
  • Cinematic card reveal — destination cards materialise from a blur/scale animation with stagger on first load
  • Aurora home screen — animated glowing orbs that drift slowly and react to mouse position
  • Comparison mode — tap the compare icon on any two cards to view them side by side
  • Show more — loads 3 additional destinations without leaving the page
  • Saved destinations — heart any destination; saved list persists with undo remove and trip notes
  • Share card — styled travel-poster modal with copy and native share options
  • Personalise panel — 5 theme presets, custom accent colour, font size, map style
Challenges

Challenges & Learnings

  • Structured AI output — designing a prompt that reliably returns a valid JSON array of destinations including coordinates, colours, and nested data
  • Per-destination colour theming — each destination has its own accent colour applied via CSS custom properties; accent updates on card hover and clears on page leave
  • 3D tilt with image parallax — combining Framer Motion's useMotionValue and useTransform to create depth without conflicting with existing hover animations
  • Result caching — hashing preferences and storing results in Supabase to avoid redundant API calls while still allowing "show more" to return fresh destinations
  • Serverless timeouts — the suggest API queries Supabase and calls Claude; raising the Vercel function limit to 60 seconds was needed to handle slow cold starts

Problem

Most travel planning tools start with a search bar and assume you already know where you want to go. I wanted to build something that starts with a conversation — letting the AI ask the right questions and surface destinations the user might never have considered.

Solution

Wander uses Claude to handle the discovery conversation, then returns three fully structured destination objects — each with coordinates for the map, a cultural colour theme for the UI, food recommendations, attractions, and practical travel info. The results page combines an interactive Mapbox map with animated destination cards and a suite of planning tools accessible from each card.

What I Built

AI

Prompt Engineering

Designed a system prompt that reliably returns a structured JSON array of destinations with all required fields — coordinates, cultural theme colours, nested food and attraction data, and practical travel info — without hallucinating or missing fields.

UI

Framer Motion Effects

Built three standout visual effects using Framer Motion: an animated aurora background on the home page, a cinematic stagger reveal when results load, and a 3D card tilt with image parallax on hover using useMotionValue and useTransform.

Feature

Planning Tools

Added three AI-powered planning features accessible from each destination: a categorised packing list, a day-by-day itinerary with morning/afternoon/evening slots, and a similar destinations finder — each backed by a separate Claude Haiku API route.

Backend

Caching & Performance

Implemented SHA-256 preference hashing to cache suggestion results in Supabase for 24 hours. The "show more" feature uses a modified hash (city exclusions added to the summary) to bypass cache and return genuinely fresh destinations.

Result

Wander is my most technically complete project — combining prompt engineering, a real database, third-party APIs, advanced animations, and a polished multi-page UI. It demonstrates what a full-stack AI product looks like in practice: not just calling an API, but building the caching, error handling, theming, and UX around it.
