RetireAI

A full-stack SaaS retirement planner that combines compound interest projections, multi-scenario analysis, and Claude-powered personalised reports — all in one dashboard.

Live Demo ↗ Source Back to Projects
Overview

Project Summary

RetireAI is a multi-page SaaS web app where users enter their financial details and get real-time projections, a scenario comparison across three return rates, and an AI report generated by Claude. The goal was to build something with real database auth, a live API integration, and a polished dashboard UI — not just a static demo.

Stack

Stack & Architecture

  • Next.js 16 — App Router, server + client components
  • TypeScript — end-to-end type safety
  • Tailwind CSS v4 — CSS-based config with custom theme variables
  • shadcn/ui — component library
  • Recharts — savings and scenario charts
  • Supabase — email/password auth and PostgreSQL database
  • Claude API — Anthropic Haiku model for AI report generation
  • Vercel — deployment
Features

Key Features

  • Savings projection chart — compound interest calculation plotted against a retirement target line
  • Scenario comparison — conservative (4%), moderate (7%), and aggressive (10%) return projections side by side
  • AI report — Claude analyses the full financial profile and returns a readiness score, bull/bear case, and 3 action items
  • Readiness score — calculated from projected savings vs. retirement gap, displayed as a live gauge
  • 5-theme picker — CSS variable injection at runtime with localStorage persistence
  • Dark mode — toggled via class on html element, persisted across sessions
  • Row Level Security — Supabase RLS ensures each user only sees their own data
Challenges

Challenges & Learnings

  • Tailwind v4 — no config file; all theming lives in globals.css using @theme blocks — a different mental model from v3
  • Recharts hydration — charts use browser APIs and break on the server; fixed with dynamic() import and ssr: false
  • API key security — the Anthropic API key must never reach the client; kept strictly server-side in a Next.js API route
  • TypeScript strict mode — Recharts formatter types and zod/react-hook-form resolver types caused build failures; fixed before deploying to Vercel
  • Mobile responsiveness — dashboard grid layouts needed breakpoint overrides to stack correctly on small screens

Problem

Most retirement calculators are either too simple (a single number with no context) or too complex to actually use. I wanted to build something that gave real projections based on a user's actual numbers, showed multiple possible futures, and used AI to turn the data into clear, actionable recommendations — all in a clean dashboard that works on mobile.

Solution

I built a multi-page SaaS app with Supabase auth and a PostgreSQL profiles table, a planner form that upserts the user's data, and a dashboard that calculates projections in real time. The AI report page sends the full profile to a server-side Claude API route and stores the result in an ai_reports table. A shared DashboardLayout handles the sidebar, mobile bottom nav, theme switching, and dark mode across all pages.

What I Built

Auth

Authentication & Database

Implemented email/password sign-up and login using Supabase Auth, with a profiles table linked by user ID and Row Level Security policies so each user's data stays private.

Dashboard

Live Projections Dashboard

Built a dashboard that loads the user's profile and calculates projected savings, monthly surplus, years to retirement, and a readiness score — all using a compound interest formula with a 7% default return rate.

Charts

Savings & Scenario Charts

Created two Recharts line charts: one showing projected vs. target savings growth over time, and one comparing all three return scenarios side by side on the same axis.

AI

Claude AI Report

Built a server-side API route that sends the user's financial profile to the Claude Haiku model with a structured prompt, then parses the JSON response into a readiness score, summary, bull/bear analysis, action items, and risk assessment — saved to Supabase for persistence.

Result

A fully deployed SaaS app with real auth, a live AI integration, and a complete multi-page dashboard — deployed on Vercel with environment variables managed securely. This was the most complex project I've built: it required connecting auth, a database, a third-party AI API, real-time calculations, and a responsive UI all in one app.
